## AI 공격의 현재

### AI 피싱: "이메일에 오타가 없다"
"맞춤법 오류를 찾아라"는 옛날 조언. AI가 LinkedIn, SNS를 분석해 5분 만에 완벽한 맞춤형 피싱을 생성한다. 클릭률 54%.

### 딥페이크: "눈과 귀를 믿을 수 없다"
딥페이크 서비스(DaaS)가 다크웹에서 상품화. 2022년 대비 **+2,137%** 증가.

### 자율 해킹 에이전트
AI가 네트워크 스캔, 취약점 발견, 공격까지 **인간 개입 없이** 자동 수행.

## AI 방어의 반격

AI 보안 도구를 사용하는 조직은 유출 당 **220만 달러를 절약**한다:

1. **실시간 위협 탐지** — 24/7 네트워크 행동 분석
2. **자동 사고 대응** — 탐지부터 격리까지 3초
3. **예측적 취약점 관리** — 공격자보다 먼저 취약점 발견
4. **딥페이크 탐지** — 음성/영상 실시간 분석

## 왜 "AI vs AI"가 구조적 전쟁인가

지금의 보안 전쟁은 단순히 공격 도구가 좋아진 수준이 아닙니다. 핵심은 **학습 속도와 자동화 범위**입니다. 과거에는 공격자가 숙련 인력과 시간이 부족해 공격 규모가 제한됐지만, 이제는 LLM 기반 자동화로 공격 시나리오를 대량 복제할 수 있습니다. 반대로 방어 측도 사람이 알림을 순차 처리하던 방식에서 벗어나, 에이전트가 우선순위를 판단하고 격리까지 수행하는 체계로 이동하고 있습니다.

즉, 승패는 "누가 더 똑똑한가"가 아니라 **누가 더 빠르게 학습하고 피드백 루프를 돌리는가**에서 갈립니다.

## 2026 공격 체인: 단계별로 무엇이 달라졌나

아래 표는 전통적 킬체인과 AI 기반 공격의 차이를 실무 관점에서 요약한 것입니다.

| 단계 | 전통 공격 | AI 기반 공격(2026) | 방어 포인트 |
|---|---|---|---|
| 정찰(Recon) | 수동 OSINT, 시간 소모 큼 | SNS/깃허브/공시정보 자동 수집·요약 | 외부 노출 정보 최소화, 허니 프로필 운영 |
| 무기화(Weaponize) | 템플릿 피싱 제작 | 개인별 문체·관계 맥락 반영한 초개인화 | DMARC/SPF/DKIM + 문맥 기반 메일 탐지 |
| 전달(Delivery) | 대량 이메일 살포 | 소량 고정밀 공격, 채널 다변화(메일+메신저+콜) | 채널별 검증 정책 통합 |
| 실행(Execution) | 사용자 클릭 의존 | 딥페이크 통화 + 즉시 송금 유도 | 고액 거래 2채널 인증, 코드워드 |
| 유지(Persistence) | 백도어 설치 | 권한 상승 자동화, 측면 이동 에이전트화 | EDR/XDR 정책 자동 격리 |
| 은닉(Evasion) | 로그 삭제/난독화 | 탐지 룰 역학습, 시간 분산 공격 | 행동 기반 이상탐지 + 위협 헌팅 |

이 변화 때문에 보안 조직은 단순 룰 추가로는 대응이 어렵습니다. 룰은 금방 우회됩니다. 결국 **행동 맥락 기반 탐지 + 자동 대응**이 기본값이 됩니다.

## 현실적인 방어 아키텍처 (사람을 제거하지 말고 재배치하라)

많은 조직이 "AI 보안 도입 = SOC 인력 감축"으로 오해합니다. 실제 우수 사례는 반대입니다. 반복 경보 처리는 자동화하고, 사람은 고위험 의사결정과 포렌식에 집중합니다.

```bash
# 권장 SOC 운영 흐름 (요약)
ingest_logs --from email,endpoint,cloud,identity
ai_correlate --window 15m --risk-threshold medium
ai_triage --classify phishing/deepfake/ransomware/exfil
if risk=high:
  auto_contain --disable-account --isolate-endpoint --block-ioc
  notify_human --channel oncall --severity critical
  start_forensics --snapshot memory,disk,network
else:
  queue_review --sla 30m
```

여기서 핵심은 `notify_human`이 사라지지 않는다는 점입니다. 자율 대응은 속도를 올리지만, 법적·운영 책임은 여전히 인간에게 있습니다.

## 딥페이크 금융 사기 대응 프로토콜

딥페이크 사건은 기술 문제이면서 동시에 **업무 프로세스 문제**입니다. 영상 통화로 임원 얼굴을 확인해도 결재하면 안 되는 이유가 여기에 있습니다.

### 최소 프로토콜
1. 고액 송금 요청은 반드시 비동기 재확인(사내 메신저/전화 역콜백).
2. "긴급"이라는 말이 들어간 요청은 자동으로 위험 점수 가중.
3. 결재자는 음성/영상 확인이 아닌 **사전 합의된 코드 문구**를 요구.
4. 실패 로그를 월간 리포트로 남기고 교육 시나리오로 재활용.

| 상황 | 허용 여부 | 이유 |
|---|---|---|
| 화상회의 단일 승인 | 금지 | 딥페이크 리스크 |
| ERP 결재 + 역콜백 1회 | 조건부 허용 | 다중 채널 검증 |
| ERP 결재 + 코드워드 + 역콜백 | 권장 | 현재 가장 현실적 |
| 법무/재무 이중 승인 | 고액 필수 | 책임 분산 |

## 개발팀이 놓치기 쉬운 보안 채무

보안 사고는 인프라팀만의 문제가 아닙니다. 개발팀이 AI를 빠르게 도입할수록 새로운 채무가 생깁니다.

- LLM 보조 코드에 하드코딩 비밀키가 남는 문제
- 테스트 통과 후 운영 환경에서 권한 과다 부여
- 프롬프트 인젝션으로 내부 도구 호출이 오작동하는 문제
- 에이전트 루프가 외부 입력을 신뢰해 반복 실행되는 문제

따라서 "기능 QA"와 별도로 **에이전트 안전성 QA**를 둬야 합니다. 예: 툴 호출 allowlist, 최대 실행 횟수 제한, 입력 정규화, 감사 로그 강제.

## 90일 실행 로드맵

보안 성숙도가 낮은 조직도 90일이면 기본 방어력을 크게 올릴 수 있습니다.

| 기간 | 해야 할 일 | 성공 지표 |
|---|---|---|
| 1~30일 | MFA 강화, 고액결재 프로토콜 도입, 피싱 훈련 업데이트 | 모의 피싱 클릭률 30%↓ |
| 31~60일 | EDR/XDR 연동, AI 위협 탐지 PoC, 계정 이상행동 탐지 | 평균 탐지시간(MTTD) 50%↓ |
| 61~90일 | 자동 격리 정책, 사고 대응 런북 자동화, 임원 대상 딥페이크 훈련 | 평균 대응시간(MTTR) 40%↓ |

수치 목표가 없는 보안 계획은 대부분 선언으로 끝납니다. 반드시 MTTD/MTTR, 재발률, 훈련 실패율 같은 지표를 함께 설계해야 합니다.

## 결론: 보안은 기술보다 운영의 문제다

AI 공격은 앞으로 더 정교해질 것이고, 방어도 자동화가 깊어질 것입니다. 그러나 마지막 방어선은 여전히 "사람이 만든 프로토콜"입니다. 아무리 좋은 탐지 모델이 있어도, 조직이 검증 루틴을 지키지 않으면 한 번의 승인으로 수십억 원이 빠져나갈 수 있습니다.

2026년 보안의 현실은 명확합니다. **AI를 도입하지 않으면 느려서 지고, 사람 절차를 버리면 빠르게 망한다.** 결국 이 전쟁의 해답은 AI와 인간 절차를 결합한 하이브리드 운영에 있습니다.
